Detailed Description

template<bool dup_pnt, double eps>
class mjr::MRPoly< dup_pnt, eps >

Template Class used to house VTK-like "polydata".

This class's primary use case is to hold VTK polydata generated from an MRPTree object, and then produce a VTK file. That said, this class is generally useful by itself so long as you can live within it's limitations.

Things to know:

  • VTK polydata generated from MRPTree objects don't require all of the complexity of VTK polydata objects
    • Only four types of cells are supported: points, segments, triangles, tetrahedrons, & pyramids.
    • Only point data is supported (i.e. no cell data)
  • VTK files produced
    • They are legacy ASCII only
    • NaN's in text VTK files are not properly handled by many VTK applications
  • Performance isn't a priority
    • Espeically true for 0-cells (stores them as a one element std::vector!
    • Geometry checks are also sub-optimal
  • This we don't check
    • Memory allocation – you run out, the thing crashes
    • Duplicate cells may be added
    • Cells that are a part of other cells may be added (i.e. a segment that is part of an existing triangle)
      Template Parameters
      dup_pntAllow duplicate points
      epsEpsilon used to detect zero

Add a point.

Baring memory issues, this function should never fail to put a new point on the master point list. Returns the index for the new point. If the point is already on the master point list, then master point list is unchanged and the index for the existing point is returned.
